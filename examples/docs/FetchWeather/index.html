activity2.js

Lab 5 activity 2 javascript template that defines the interaction between this client and the www.apixu.com weather API. It uses a series of real-time fetching methods and graphical information to display current weather information in a give set of regions using a dynamic table platform.

Key required to fetch weather data from Apixu

INIT_CITIES

Defines a list of cities to populate to the UI select box. The cities can be selected and have their real-time weather information fetched and displayed.

INIT_CITIES

weatherConditions

Stores weather conditions and their corresponding descriptions, codes, and other information (for complete listing of codes see Apixu weather conditions)

weatherConditions

average

Stores the average temperature of the list of cities

average

hottest

Stores the city name and temperature of the hottest city amongst the list of cities

hottest

setLayout

Sets the initial layout of the weather page by:

  • Setting the average and hottest temperatures on page header
  • Drawing and populating table information
  • Setting 'Forecast' button event listeners
setLayout()

populateCity

Populates the table with a city value using a series of parameters to identify the correct table position

populateCity(trId: string, city: string, typeOfReport: string)
Parameters
trId (string) The id of the 'tr' object in the city-table table
city (string) The city to populate the record information with
typeOfReport (string) Current or forecast report to pull from ther apixu weather API

updateTableCity

Defines the event listener for the select box, which allows the user to select a value and the table to reflect with the newly-chosen city information

updateTableCity()

fetchCityAPI

Fetches JSON data from the server based on the type of report requested and the city.

fetchCityAPI(typeOfReport: string, selectCity: string): any
Parameters
typeOfReport (string) 'current' or 'forecast' report
selectCity (string) The city being fetched
Returns
any: The apixu JSON API as a Promise

fetchWeatherConditions

Fetches the apixu weather conditions with a list of codes and their corresponding night/day descriptions. Drawn from http://www.apixu.com/doc/Apixu_weather_conditions.json

fetchWeatherConditions()

handleError

Handles and displays errors if passed by the apixu JSON API

handleError(error: number)
Parameters
error (number) The JSON error object as a Promise

getHTTPStatusCode

Returns the HTTP status code based on the error code defined by the apixu error API. See: https://www.apixu.com/doc/errors.aspx

getHTTPStatusCode(code: number): any
Parameters
code (number) The apixu error code
Returns
any: The corresponding HTTP status code

getForecast

onclick event listener for the 'Forecast' buttons on the weather table.

getForecast(trId: string)
Parameters
trId (string) The id of the 'tr' to draw the city info from

setForecastField

Sets the forecast div ('forecast-info') using response data generated from apixu

setForecastField(response: JSON)
Parameters
response (JSON) Forecast data fetched from apixu

getWeatherCodeDescription

Returns the corresponding weather code to the passed parameter. The weather codes are drawn from: http://www.apixu.com/doc/Apixu_weather_conditions.json

getWeatherCodeDescription(code: number): any
Parameters
code (number) Weather status code
Returns
any: JSON representation of the weather code. Null if there is no corresponding code in the apixu API

getForecastButtonstring(trId: number): any

Returns
any: A button object to be inserted into the DOM

getSelectedCity

Returns the selected city of the city-select drop down

getSelectedCity(): any

any: the selected city of the city-select drop down
